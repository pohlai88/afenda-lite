## Pipeline

<Steps>
<Step>

### Regenerate OpenAPI

From the monorepo root, after handler or Zod changes:

```bash
pnpm openapi:generate
pnpm check:openapi
```

</Step>
<Step>

### Refresh API MDX

```bash
pnpm --filter @afenda/docs generate:openapi-docs
```

Hand-maintained `content/docs/api/index.mdx` must stay; operation pages and
`api/meta.json` come from `generateFiles`.

</Step>
<Step>

### Source, links, build

```bash
pnpm --filter @afenda/docs generate:source
pnpm --filter @afenda/docs lint:links
pnpm --filter @afenda/docs typecheck
pnpm --filter @afenda/docs build
```

Lean gate without a full Next build: `pnpm check:docs-app`.

</Step>
</Steps>

## Pitfalls

<Accordions type="single">
  <Accordion title="Sidebar missing after a new page">
    Append the slug to the nearest `meta.json` `pages` array, then
    `generate:source` and `lint:links`.
  </Accordion>
  <Accordion title="Hand-edited operation MDX vanished">
    Operation pages under `api/` are owned by `generate:openapi-docs`. Edit the
    Zod/OpenAPI source instead, then regenerate.
  </Accordion>
  <Accordion title="Search returns nothing">
    Confirm `app/api/search/route.ts` exports `GET` from `createFromSource(source)`
    and that `generate:source` has run so the page tree is current.
  </Accordion>
</Accordions>

## Add a narrative page

1. Add `content/docs/<slug>.mdx` with `title` and `description` frontmatter.
2. Append `<slug>` to `content/docs/meta.json` (or a folder `meta.json`).
3. Run `pnpm --filter @afenda/docs generate:source` and `lint:links`.
4. Verify `/docs/<slug>` on port **3001**.
